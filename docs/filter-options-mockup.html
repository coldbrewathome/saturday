Filter Redesign Options

The browse view searches and filters both spots and events, but the current UI doesn't make that clear. Here are three approaches to fix the ambiguity.

Current Problem

The filter panel mixes spot-only and event-only filters with no visual separation. Users may think all filters apply to everything. The search placeholder says "Search spots" but also searches events.

Option A
Tabbed Filter Panels

A Spots / Events toggle at the top of the panel. Each tab shows only the filters that apply to that type. Map and list update to show only the selected type.

Pros
  • Zero ambiguity — every filter works
  • Cleaner, shorter panel per tab
  • Easy to add type-specific filters later
Cons
  • Can't see both types at once
  • Extra tap to switch contexts
  • Users may miss events (or spots)

Option B
Grouped Sections

One unified panel with labeled sections: Shared filters on top, then Spots and Events sections below. Both types always visible on the map.

Pros
  • Everything visible at once
  • Clear which filters affect what
  • No mode switching needed
Cons
  • Panel is longer / more visual clutter
  • Sections may feel bureaucratic
  • May overwhelm casual users

Option C
Progressive Disclosure

A All / Spots / Events segmented control. "All" shows only shared filters. Picking a type reveals type-specific filters and scopes the map.

Pros
  • Clean default with minimal filters
  • Progressive disclosure — simple → detailed
  • Placeholder text stays accurate
Cons
  • "All" mode has fewer filter options
  • May feel limited until users discover tabs
  • Three states to maintain
